\section{Describing Structtures Beyond Classic Linked Lists}

Separation logic also allows us to describe complex structures, such as N-arty trees:

\begin{alignat*}{3}
tree(x,c) &\quad \on{def}{=}\quad && (x \mapsto 0,\_ \star c = 1)\\
&\quad \quad && \vee \left(\exists\   n > 1, \vec{p}, \vec{c}. x \mapsto n \star c = \sum_{i = 1}^n c_i \star P(n,c) \right)\\
P(n,c) &\quad = \quad &&  \bigcircledast_{1 \leq i \leq n} (x+i \mapsto p_i \star tree(p_i,c_i) \wedge 1 \leq c_i \leq c) )
\end{alignat*}

The predicate $tree(x,c)$ describes a tree $x$ with $c$ total cells used by either constructing a leaf with value $0$ (and an empty box for uniformtiy), or by stablishing what the head $x$ is pointing to, and ensuring we have enough space for each of the children ($c_i$) and describing them recutrsively.

\section{Formalizing Proof Sketches}

In the previous lecture we saw how we could describe informally the behaviour of a program, such as \verb!LLen! (which comptuted the length of a linked list recursively) via Hoare Triplets with pre and post conditions encoded in separation logic. But how can we reason about the program in a formal way?

For that, we'll use the proof rules defined in Figure \ref{fig:proof_rules} (\textbf{\textcolor{red}{frame}}, \textbfr{cons}equence, \textbfr{exist}ential elimination, and \textbfr{seq}uencing) to do proof derivations.

\begin{figure}[H]
\[
\begin{prooftree}
\hypo{\vdash \hoareT{P}{C}{Q}}
\hypo{mod(C) \cap fv(R) = \emptyset}
\infer2[frame]{\vdash \hoareT{P \star R}{C}{Q \star R}}
\end{prooftree}
\qquad 
\begin{prooftree}
\hypo{\models P \Rightarrow P'}
\hypo{\vdash \hoareT{P'}{C}{Q'}}
\hypo{\models Q' \Rightarrow Q}
\infer3[cons]{\hoareT{P}{C}{Q}}
\end{prooftree}
\]
\[
\begin{prooftree}
\hypo{\vdash \hoareT{P}{C}{Q}}
\infer1[exists]{\vdash \hoareT{\exists x. P}{C}{\exists x. Q}}
\end{prooftree}
\quad
\begin{prooftree}
\hypo{\vdash \hoareT{P}{C_1}{Q}}
\hypo{\vdash \hoareT{Q}{C_2}{R}}
\infer2[seq]{\vdash \hoareT{P}{C_1;C_2}{R}}
\end{prooftree}
\]
\label{fig:proof_rules}
\caption{Proof rules for reasoning about Hoare Triplets}
\end{figure}

\textbfr{Notation}: $mod : C \rightarrow Var$ is a function which maps commands to the (program) variables which are (possiblty) modified, while $fv$ returns the set of free variables (program or logical) from an assertion expression.
